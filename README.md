###### Подготовка сервиса к запуску

Создать `.env` из примера и заполнить переменные:
```
cp .env.example .env
```

###### Порядок первого запуска

```
make init
```

Команда последовательно выполнит: `keys` → `build` → `up` → `migrate` → `superuser`

###### Команды

Создать приватный и публичный ключи:
```
make keys
```

Собрать образ:
```
make build
```

Запустить сервисы:
```
make up
```

Применить миграции:
```
make migrate
```

Создать суперпользователя (потребуется ввести email, пароль):
```
make superuser
```

Остановить сервисы:
```
make down
```

Логи приложения:
```
make logs
```

Создать новую миграцию:
```
make revision name=описание
```
